# 收割文件标准格式 v3.0

> 借鉴 Cline Memory Bank 6 文件结构。每次 `/总结` 生成的收割文件遵循此格式。

---

## 文件路径

```
{SKILL_DIR}/harvests/{project}/YYYY-MM-DD-{session-id}.md
```

| 变量 | 说明 |
|------|------|
| `{project}` | 项目名，从当前工作目录提取 |
| `{session-id}` | 会话标识，从对话首条时间戳或序号生成 |

---

## 文件模板（Memory Bank 结构）

```markdown
# 收割 — {project} — {YYYY-MM-DD HH:MM}

> 会话ID: {session-id} | 轮次: ~{N} | 压力: 🟢/🟡/🔴
> 错误: {N} 条（全局 {G} / 项目 {P}）

---

- **最近决策**: {最近做的方向性选择}

## 与 Cline Memory Bank 的映射

| Cline 文件 | 我们的分区 | 说明 |
|-----------|----------|------|
| `activeContext.md` | `activeContext` | 当前焦点 + 最近决策 |
| `progress.md` | `progress` | 完成/进行/待做 |
| `decisionLog.md` | `decisions` | 决策 + 原因 + 备选 |
| _(无直接对应)_ | `errors` | 错误(全局/项目) + 修复 + 预防 |
| `systemPatterns.md` | `patterns` | 发现的模式 + 偏好 |
| _(无直接对应)_ | `回测摘要` | 规则遵守验证（v3.0 新增） |
| _(无直接对应)_ | `nextSteps` | 建议（非待办） |

```
