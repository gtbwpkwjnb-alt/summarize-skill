# 规则模板 v3.0

> `/总结` 检测到应沉淀规则时，按此格式生成。支持三级作用域分流。

---

## 规则卡片格式

```markdown
### [{规则ID}] {规则标题}
- **触发场景**: {描述什么情况下触发此规则}
- **建议行为**: {AI 应该怎么做}
- **优先级**: P0/P1/P2
- **作用域**: 全局 / 项目({name}) / 代码
- **来源**: {哪个 ERROR/DECISION 触发了此规则}
- **出现频次**: {N} 次 / {M} 个会话
- **状态**: 待回测验证
```

---

## 优先级定义

| 优先级 | 含义 | 示例 |
|:-----:|------|------|
| P0 | 阻断性 — 不遵守会导致错误结果 | 编造 API 名称、Edit前未Read |
| P1 | 高影响 — 显著降低效率或质量 | 不先搜GitHub就连续试错 |
| P2 | 改进型 — 锦上添花 | 可选的格式规范 |

---

## 作用域判断标准

| 作用域 | 写入位置 | 判断条件 |
|:-----:|------|------|
| **全局** | `AGENTS.md`（工作区根） | 错误涉及工具使用方式/通用流程，任何项目都会遇到 |
| **项目** | `{project}/.zcode/AGENTS.md` | 错误涉及特定项目配置/架构/领域知识 |
| **代码** | 不存为规则 | 错误涉及代码实现细节 → 转为代码优化建议 |

**自动判断逻辑**：
- 错误分类为 PROC/TOOL → 通常是全局
- 错误分类为 KNOW → 通常是项目或代码
- 错误涉及特定文件路径/项目名 → 项目
- 同类型错误跨 2+ 项目出现 → 全局

---

## 规则建议触发条件

`/总结` 自动判断是否建议新规则：

1. **同类型错误出现 3+ 次（跨会话）** → 必须建议 P0/P1 规则
2. **同类型错误出现 2 次（同会话）** → 建议 P2 规则
3. **用户反复表达相同偏好 2+ 次** → 建议偏好规则
4. **出现新的环境约束或工具限制** → 建议环境规则
5. **回测发现规则反复被违反** → 建议修订现有规则

```markdown

| Edit前未Read | 4 | PROC | 06-10 | 06-16 | 1 | P0-Edit前Read | ❌ | ZCode | ⚠️⚠️⚠️ |
```

```markdown

|---------|:---:|:--:|------|------|:--------:|---------|:-----:|------|
| CC-Switch冲突 | 1 | KNOW | 06-17 | 06-17 | 0 | P1-约束矛盾指出 | ❌ | 🆕 |
```
