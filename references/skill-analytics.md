# 技能调用统计详细规范

## Token 数据来源

| 优先级 | 来源 | 精度 |
|:--:|------|:--:|
| 1 | ZCode 上下文窗口使用率（输入框下方统计） | 精确 |
| 2 | 字符估算（中文 ~1.5 字符/token，英文 ~4 字符/token） | 估算 `~` |
| 3 | 无数据 → 仅显示调用次数，token 列留空 | - |

## 效果标记

| 标记 | 含义 | 示例 |
|:--:|------|------|
| ✅ | 节省 token | summarize 压缩上下文 |
| ⚠️ | 消耗 token | skills-audit 完整审计 |
| ➡️ | 中性 | 工具性调用 |

## 输出格式

```
📊 技能调用:
| {技能名} | {N}次 | token ~{估算} | {效果}
```

### 示例（有 token 数据）

```
📊 技能调用:
| summarize | 3次 | token ~1,200 | ✅ 节省
| skills-audit | 1次 | token ~8,500 | ⚠️ 消耗
| agent-reach | 2次 | token ~600 | ➡️ 中性
```

### 示例（无 token 数据，降级）

```
📊 技能调用:
| summarize | 3次 | token — | ✅
| skills-audit | 1次 | token — | ⚠️
```

## 写入 _self-stats.md

每次 `/总结` 将技能调用数据追加到 `harvests/_self-stats.md`：

```
## 技能调用记录 — 2026-06-28T20:00:00
| 技能 | 调用次数 | token估算 | 效果 |
|------|:--:|:--:|:--:|
| summarize | 3 | ~1,200 | ✅ |
| skills-audit | 1 | ~8,500 | ⚠️ |
```

同一天多次总结 → 合并同一天的数据行，更新调用次数和 token 总和。
