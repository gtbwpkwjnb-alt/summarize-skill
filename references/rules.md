# 进化规则 v5.0 — 三级作用域（通用版）

> 由 `总结` 模块3自动沉淀。跨会话复用的行为规则。
> 这些是你当前平台的示例规则。初次安装后模块3会自动生成适合你的规则。

---

## P0 — 强制执行（示例）

| 规则 | 触发 | 行为 |
|------|------|------|
| P0-1: 先查协议再写mock | 未知网络端点 | 确认传输层协议后再写代码 |
| P0-2: 同工具受阻3次换工具 | 围绕同一工具反复补丁 | 第3次受阻时退一步评估工具适用性 |
| P0-3: 连失2次换方法 | 同一思路失败2次 | 第3次换思路，失败时必须说出受阻原因+替代方案 |
| P0-4: 对外网不可达时用SSH | git push/clone到外网仓库 | 优先 SSH，备选 API 代理 |
| P0-5: 改文件前读原内容 | 任何改文件操作 | 确保已读取原文件再执行修改 |

## P1 — 强烈建议（示例）

| 规则 | 触发 | 行为 |
|------|------|------|
| P1-1: 第2轮试错前搜同类报告 | 非trivial报错 | 搜 GitHub Issues/Discussions 确认 |
| P1-2: 约束矛盾当场指出 | 用户约束互斥 | 动手前指出矛盾 |
| P1-3: 不编造未确认的标识符 | 引用函数/API/路径前 | 先用搜索确认存在再引用 |

> 这些是初始示例规则。模块3会根据你的实际错误模式自动沉淀、更新和优化这些规则。
