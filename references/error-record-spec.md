# 错误记录字段规范 / Error Record Field Specification

> 每条错误在账本（`error-ledger.md` / `{project}/errors.md`）中的标准格式。

---

## 标准表头

```markdown
| 错误类型 | 次数 | 分类 | 作用域 | 首次 | 最近 | days_clean | 避免规则 | 已验证 | 涉及项目 |
|---------|:---:|:--:|:-----:|------|------|:--------:|---------|:-----:|---------|
| {类型} | {N} | {PROC/ASSU/ENVR/TOOL/KNOW} | 全局/项目 | YYYY-MM-DD | YYYY-MM-DD | {N} | {规则ID} | ✅/❌ | {项目} |
```

## 字段说明

| 字段 | 说明 |
|------|------|
| `错误类型` | 简短描述，如 "Edit前未Read"、"GitHub HTTPS被墙" |
| `次数` | 累计触发次数 |
| `分类` | PROC / ASSU / ENVR / TOOL / KNOW（见模块 2.1） |
| `作用域` | `全局`（跨项目复用）或 `项目`（特定项目） |
| `首次` | 首次出现日期 |
| `最近` | 最近出现日期 |
| `days_clean` | 自上次出现后连续未再犯的天数（正向激励，复发归零） |
| `避免规则` | 对应的预防规则 ID，如 `P0-Edit前Read` |
| `已验证` | 该规则是否经回测确认有效（✅/❌） |
| `涉及项目` | 出现过的项目列表，逗号分隔 |
