## P0 — 强制执行（全局）

### P0-1: 模拟协议前先查协议规范
- **触发**: 遇到未知网络端点，想写 mock/tunnel 时
- **行为**: 先查该端点的传输层协议（HTTP REST? JSON-RPC? WebSocket? gRPC?），确认后再写代码
- **反例**: 看到 `chatgpt.com/backend-api/wham/apps` 返回 JSON 就写 HTTP mock，实际是 JSON-RPC over WebSocket
- **来源**: codex/2026-06-17 | **回测**: ❌ 未验证 | **干净天数**: 0

### P0-2: 同一工具受阻 3 次 → 换工具，不打补丁
- **触发**: 围绕同一工具的设计行为反复修改配置/打补丁
- **行为**: 第 3 次受阻时退一步，问"这个工具适合当前场景吗？"，而非继续修
- **反例**: CC-Switch takeover 反复覆盖 config.toml，试了 hosts 文件、只读、改 DB、禁用 live_takeover 4 种方法对抗
- **来源**: codex/2026-06-17 | **回测**: ❌ 未验证 | **干净天数**: 0

### P0-3: 连失 2 次换方法
- **触发**: 同一思路尝试 2 次均失败

### P1-1: 第 2 轮试错前搜 GitHub issues
- **触发**: 遇到非 trivial 报错，本地试了 1 轮没解决
- **行为**: 第 2 轮尝试前先搜 `repo:xxx issues` + WebSearch，确认是否有同类报告
