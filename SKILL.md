# 总结 v5.3 — 精炼 · 进度 · 自进化 · 可逆 · 跨Agent（多平台通用）

> **设计目标**: 任务开发>1天时，让你扫一眼就了解会话全貌。
> **核心原则**: 一句话能表达清楚绝不用两句。不得输出与4模块无关的内容。
> **平台**: ZCode / Claude Code / Codex / Cursor / Windsurf / 任何支持自定义技能的 AI 编程助手

---

## 触发方式

**唯一触发词**：`总结` 或 `summarize`，且必须作为独立词单独发送。句中出现的"总结"不触发。无自动提醒、无定时触发、无其它激活路径。

| ✅ 会触发 | ❌ 不会触发 |
|-----------|-------------|
| `总结`（单独发送） | "总结一下今天的工作"（在句中） |
| `summarize`（单独发送） | "帮我总结一下进度"（在句中） |
| `总结 统计`（独立词+参数） | 会话轮次/错误数达到阈值（无自动提醒） |

### 产出可能为空的场景

以下场景技能仍会触发，但部分模块可能无产出（不是不触发）：

| 场景 | 影响 |
|------|------|
| 会话<5轮且无错误 | 模块3无可收割内容 |
| 纯对话无工具调用 | 模块1文件清单为空 |
| 同一会话10轮内已执行过 | 增量不足，摘要变化小 |

## 输出精炼规则

> 每个模块有硬上限。超出部分折叠为一行 `📦 详见归档`，完整内容写入 `harvests/{project}/{session-id}.md`。

| 模块 | 上限 | 超出处理 |
|------|------|---------|
| 模块1 文件清单 | 8个文件 | `…等{N}个文件 📦` |
| 模块1 关键决策 | 5条 | `…等{N}条决策 📦` |
| 模块3 错误列表 | 5条（按次数降序） | `…等{N}条错误 📦` |
| 整体输出 | ≤20行 | 超限时优先折叠模块3→模块1 |

## 归档模型

> 每个会话一个归档文件，每次 `总结` **覆盖更新**（非追加）。避免数据重复，保持精炼。

```
harvests/
  error-ledger.md              ← 全局错误（跨项目复用）
  _self-stats.md               ← 技能自身运行统计
  index.md                     ← 收割索引（同会话只保留最新一条）
  {project}/
    errors.md                  ← 项目错误
    {session-id}.md            ← 当前会话归档（覆盖更新）
```

**归档文件内容** (`harvests/{project}/{session-id}.md`)：
```markdown
# {项目名} — {session-id} 归档
> 最后更新: {时间戳}

## 压缩上下文（跨会话交接）
**目标**: {1句任务目标}
**当前状态**: {1句当前进度}
**关键决策**: {决策列表，每条附带原文锚定引用}
**使用技能**: {本次会话调用的技能及次数}
**产出物**: {文件/代码变更清单}

## 完整文件清单
{所有涉文件，不截断}

## 完整错误列表
> 每条错误附带原文锚定引用，防止反向理解偏差。

| 错误 | 分类 | 原文引用 |
|------|:--:|---------|
{完整错误列表，不截断，每条附带原文锚定}

## 技能调用详情
{每次技能调用的时间、token估算、产出}
```

**更新策略**：
- 每次 `总结` → 覆盖写入同一文件（文件头更新 `最后更新` 时间戳）
- `index.md` 同会话只保留最新一条索引记录
- 旧版 `{date}-{session}.md` 格式文件在下次总结时自动迁移到 `{session-id}.md`

## 模块 1：会话精炼（核心）

**任务**: 会话→≤5句关键摘要 + 涉文件清单

```
📋 {项目} —— {1句总结}
> {3-5句关键摘要}
📁 文件({N}): {前8个路径}({操作}: {函数/类名}), …等{N}个文件 📦
   （超8个时折叠，完整清单见归档。函数/类标注可选——无git/非代码项目降级为纯路径）

🔑 关键决策({N}):
- {前5条决策}
   （超5条时折叠，完整清单见归档）
```

🖼️ 图片降级: 会话含图片时生成文本描述替代原图引用（需平台支持多模态访问）；无图片或不可访问则跳过。

## 模块 3：错误自进化（核心）

> 执行前加载 AGENTS.md 的 `<error_patterns>` 表作为规则源，错误历史统计见 `harvests/error-ledger.md`。规则 ID 必须与 AGENTS.md 一致。

**检测**: 工具error / 用户纠正 / 同一操作重试≥3次

**5 维分类**（通用版，按你的平台替换例子）：

| 代码 | 分类 | 说明 | 示例 |
|:----:|------|------|------|
| PROC | 流程违规 | 跳过必要步骤 | 改文件前没读原内容 |
| ASSU | 假设错误 | 用了未确认的标识符/路径 | 编造不存在的函数名 |
| ENVR | 环境问题 | 网络/系统/权限限制 | 外网不可达，命令不支持 |
| TOOL | 工具误用 | 选了不适当的工具 | 用通用命令代替专用工具 |
| KNOW | 知识盲区 | 缺乏领域/项目上下文 | 不熟悉项目约定 |

```
⚠️ 错误({N}):
| {前5条错误} | {分类} | {N}次 | {标记}
  …等{N}条错误 📦（超5条时折叠）

🛡️ 规则:
✅ {规则} 干净+{N}
⚠️ {规则} 违反
🔄 {规则} 复发——修订建议:{行动}

⚡ 进化:
全局→ {规则建议}
项目→ {规则建议}
```

**标记**: 🆕首次 / ⚠️2次 / ⚠️⚠️⚠️≥3次

## 模块 4：技能调用统计（新增）

> 统计当前项目中用户级技能的调用次数和效果。仅统计技能（summarize/skills-audit 等），不含底层工具（Read/Write 等）。

**Token 数据来源**（优先级）：
1. 从 AI 助手的上下文窗口统计中提取使用率（通常显示在输入框附近）
2. 无数据时用字符估算：中文 ~1.5 字符/token，英文 ~4 字符/token
3. 估算值标记 `~`

```
📊 技能调用:
| {技能名} | {N}次 | token ~{估算} | {效果}
```

**效果标记**：
| 标记 | 含义 | 示例 |
|:--:|------|------|
| ✅ | 节省 token | summarize 压缩上下文，净省 token |
| ⚠️ | 消耗 token | skills-audit 完整审计，净耗 token |
| ➡️ | 中性 | 工具性调用，无明显 token 影响 |

**详细数据**（每次调用的时间、token估算、产出）写入归档文件 `技能调用详情` 节。

## 格式原则

- **先结论后过程**: 每个模块先说结果
- **一行一事**: 不堆砌
- **上限折叠**: 超模块上限自动折叠→归档，保持内联≤20行
- **高频高亮**: ≥3次 ⚠️⚠️⚠️
- **收割内聚**: 所有数据在 `harvests/` 内
- **归档覆盖**: 同会话归档文件覆盖更新，不追加
- **表格优先**: 归档数据用表格格式，避免 JSON 代码块（减少 token 开销）
- **建议非待办**: 用户自主选择
